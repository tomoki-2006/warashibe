# 01_GDD — ゲームデザインドキュメント（完全定義版）
*Status: v3.0 / Phase 0スコープ確定。数値はすべて実装値（変更時は本書を先に更新）*

# 1. ゲームフロー全体（状態遷移の正）

```
[Title]
  └ はじめる/つづきから
[MapTravel] ←──────────────┐
  └ ストップ到着            │
[Encounter]                 │
  ├ はなす → [Dialogue]     │
  ├ しつもん → [Question]   │
  ├ こうかん → [OfferSelect]│
  └ にもつ → [Inventory]    │
[OfferSelect]               │
  └ アイテム選択 → [OfferResolve]
[OfferResolve]              │
  ├ 成立 → [TradeAccept] → [ValueMeter] → [StairUp] ─┐
  └ 拒否 → [PoliteDecline(+hint)] → [Encounter]      │
                            │                         │
        次ストップ解放 ──────┘←────────────────────────┘
[RouteClear]（最終ストップ成立時）
  └ [ChainAnimation] → [ResultCard] → もういちど/タイトルへ
```

- どの状態でも「にもつ」「ちず」は開ける（モーダル）。会話中の地図移動は不可（confirm後に会話破棄で可）
- 中断・復帰: 全状態遷移直後にオートセーブ。復帰は直前の安定状態（Encounter先頭）から

# 2. 交換ルール（Exchange Rules）

## 2.1 判定仕様
- NPCの `accepts[]` に、提示アイテムIDが含まれる → 成立
- 含まれない → `polite_decline`＋ヒント梯子を1段進める
- **同一NPCへの同一アイテム再提示は選択UIで無効化（グレーアウト＋ぶんの一言「それはさっき見せたよ」）**
- 複数アイテム同時提示は不可（Phase 0）。あわせ技は事前にインベントリで合成してから

## 2.2 あわせ技（Recipe）
- インベントリで2つ選択→「くみあわせる」
- レシピ一致→合成演出(1.2s)→新アイテム獲得、素材は消滅
- 不一致→ぶん「うーん、なにも起きないみたい」（ペナルティなし・回数無制限）
- Phase 0のレシピは1つのみ: `wara + abu = abumushi_toy`

## 2.3 質問（Question）
- 各NPC最大2問。使い切ると「しつもん」ボタンは非表示（グレーではなく消す。押せない物を見せない原則）
- 質問はスコアに影響しない（観察を促す無料リソース）

# 3. ヒント梯子（Hint Ladder）仕様

| 段 | 発火条件 | 出し方 |
|---|---|---|
| L1 | 1回目の拒否直後 | NPCの断り文に内包（例「いま欲しいのは、もっと…」） |
| L2 | 2回目の拒否直後 | ぶんの吹き出し（画面右下、tap-to-dismiss） |
| L3 | **3回目の拒否直後のみ（時間経過では発火しない・v3.2）** | ぶんが実質答えを言う＋対象を光らせる |
| nudge | 滞在45秒無操作 | ぶんの相槌のみ（「ぶんぶん…よく見てみよ？」ヒント段は進めない） |

- 滞在タイマーはEncounter入場でリセット。モーダル(図鑑等)表示中は停止
- L3後も自動では進まない。**最後のタップは必ず子ども自身**（達成の帰属を奪わない）
- deepestHint（スコア反映）は拒否経由でのみ更新。**ゆっくり考えることは絶対に減点しない**（v3.2: 90秒自動L3は「考える時間を奪わない」原則と矛盾するため廃止）

# 4. 目利き度（スコア）数式

```
ストップ得点 = base(100)
  - 質問使用: 上限 -30（1問 -15）
  - ヒント到達: L1 -30 / L2 -50 / L3 -80  ※最深到達のみ適用（累積しない)
  - 下限: 10（成立すれば必ず正の得点）
旅スコア = Σストップ得点（採点対象6ストップ・最大600。ストップ0は採点外）
  + 観察ボーナス: observableタップ後の初回成立 +10/ストップ（表示は「よくみたね！」のみ・v3.2）
```

| ランク | 閾値 | 演出 |
|---|---|---|
| 長者 | ≥540 | 金の暖簾＋紙吹雪（質問2回までなら到達可＝「質問は無料」の思想と整合） |
| 大商人 | ≥420 | 朱の暖簾 |
| 行商人 | ≥250 | 藍の暖簾 |
| 見習い | <250 | 生成りの暖簾＋ぶん「つぎはもっと上手くなるよ！」 |

- **減点は画面に一切出さない。** 表示は加点演出と最終ランクのみ（数式は内部処理）
- スコアが低くても解放・報酬の差はゼロ（Phase 0）。ランクは再挑戦動機のみに使う

# 5. 価値メーター仕様

- 表示タイミング: TradeAccept演出の直後、自動表示(スキップ不可・2.5s)
- 内容: 譲ったアイテムについて「あなたにとって★n」「あいてにとって★m」＋相手の一言理由
- ★の値は `items.json` の `baseValue`（あなた側）と `npcs.json` の `valueForNpc`（相手側）を使用
- **必ず m > n になるようデータを設計する**（CI検証項目: 全成立取引で valueForNpc > baseValue）
- 「価値の階段」は**gives連鎖の序数**で上がる（baseValueは価値メーター表示専用）。postEventで強化される取引（弱った馬）は半段点滅→世話完了で1段確定（v3.2）

# 6. 移動・マップ仕様

- 横スクロール絵巻。解放済みストップのみタップ可能（未解放は靄）
- 移動演出: 旅人＋ぶんが歩くループアニメ2.0s（スキップ: タップで0.5s）
- Phase 0は一本道（後戻り可・ただし再訪NPCは「がんばってな！」の定型のみ）

# 7. チュートリアル（ストップ0〜1が兼ねる）

| 教える操作 | 教える場所 | 方法 |
|---|---|---|
| タップ移動 | S0 | おばあさん「あの鳥居まで行ってみんさい」＋目標が明滅 |
| はなす | S0 | 近づくと自動で会話開始（初回のみ） |
| にもつ確認 | S0 | 藁入手直後、にもつボタンが1回だけバウンス |
| しつもん | S1 | 母親の初回セリフ後、ぶん「聞いてみたら？」 |
| あわせ技 | S1 | アブ捕獲後、インベントリを自動で開き2枠が明滅 |
| こうかん | S1 | おもちゃ完成後、こうかんボタンが明滅 |

- 文字での操作説明パネルは全面禁止。すべて状況＋明滅＋ぶんの一言で誘導

# 8. アブ捕獲ミニゲーム（S1内）

- 画面内をゆっくり8の字飛行するアブを3回タップで捕獲（1タップごとに減速）
- 失敗なし・時間制限なし。当たり判定はアブ絵の2倍（3〜6歳の指でも成功する寛容判定）
- 捕獲成功: 「つかまえた！」＋ぶん初登場の名乗り（「ぶん、っていうんだ。つれてって！」）→ 以後常時随伴

# 9. エッジケース定義

| ケース | 挙動 |
|---|---|
| ルート途中でアプリ終了 | オートセーブから復帰（Encounter先頭） |
| 「もういちど」選択 | インベントリ初期化・図鑑/クリア記録は保持・スコアはベスト更新方式 |
| つづきから でセーブなし | ボタン非表示 |
| localStorage破損 | version不一致/parse失敗→初期化＋「旅の記録が見つからなかったので、はじめから」ダイアログ |
| オフライン | Phase 0は全機能オフライン動作（PWA/キャッシュ済みなら機内モードでも完走可能）— 移動中プレイ・学校の回線事情への回答 |
| 連打・多重タップ | 全ボタン300msデバウンス。演出中は入力ロック |

# 10. 結果カード仕様

- 生成: RouteClear時にクライアント側で1080×1350px PNG（縦・SNS最適比率）を描画
- 記載: 成長チェーン絵文字列／ニックネーム＋ルート名／目利きランク（暖簾色）／交換回数／`#わらしべワールド`／ぶんが右下に必ずいる
- **禁止: 実名・日付・位置情報・写真。** ニックネームは表示切替可（親ゲート内設定）
- 共有: OSシェアシート（Web Share API）。非対応ブラウザは画像保存ボタンにフォールバック
