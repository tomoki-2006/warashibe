# 10_ENDGAME_AND_VISION — ゴールの先の物語と10年ビジョン
*Status: v3.1 / 「クリアの先」と「事業の先」を設計する。失敗の先回り（プリモーテム）を含む*

# 1. ゲーム内: 屋敷の先の物語（クリア後アーク）

## 1.1 なぜ「先」が要るか
「藁→屋敷」は完結が美しいがゆえに、達成した子の目的が消える。教育ゲームの死因の多くは「クリア後の空白」。そこで本作は、クリアを終点ではなく**人格の転換点**として設計する。

## 1.2 三幕構成の第二幕: 「与える長者」への転身
屋敷を得た春、大店の主人が帰ってくる。そして言う——
「やしきは そなたの ものじゃ。だが ちょうじゃに なった ものには、つぎの しごとが ある。**こんどは、だれかの『わら』に なって やることじゃ。**」

ここからクリア後コンテンツの軸が「得る」から「与える」に反転する:

- **寄進システム【Phase 3】**: 稼いだ富で町に橋・井戸・寺子屋を建てられる。建てた設備は自分の世界の町を恒久的に発展させ、住民のセリフが変わる（「あの はしの おかげで、いちばへ いきやすく なったよ」）。**富の最終形は「みんなが使うもの」**——ピックアックス理論の子ども版であり、原作わらしべ長者が観音様への感謝で締まる精神の現代化
- **恩送りゴースト（08 D-1）がここで解放**: 与える長者になった者だけが、他の子の世界に「旅の先輩」として現れる資格を得る。「あなたのゴーストが、きょう、どこかの たびびとを たすけました。恩送り +1」
- **のれんの時代（最終・第6の時代）【Phase 4】**: 物々交換→通貨→銀行→株→信用の先、最後に解放される時代の答えは「贈与と信頼の経済」。数値の資産を最大化した先に、「助けた人の数」「育てた町」「送った恩」だけが表示される画面がある。**このゲームが子どもに残す最後の一文: 「いちばん たかい ものは、しんらいと ありがとう でした」**

## 1.3 第三幕: 新しい藁（次の旅への橋）
与える長者の物語が一段落すると、旅立ちの朝が来る。荷物は持たない。国境で一本の藁を拾って、次の地方（ルート2以降・将来は海外）が始まる。
- 全ルートがこの「裸一貫の再出発」で接続され、**熟達だけが持ち越される**（アイテムはリセット・目利きと知識は残る)。「本当の資産は腕と信用」というメッセージが構造で語られる

## 1.4 クリア後の遊びの全体像（リテンションの最終形）

| 動機 | 受け皿 |
|---|---|
| もっと上手く | しばりたび・目利きランク更新（08 A-3） |
| もっと集めたい | 図鑑・ぶんの巣・きせつの市 |
| もっと広く | 次ルート・時代解放・海外 |
| 誰かのために | 寄進・恩送りゴースト・現実ミッション |

# 2. 事業の先: 10年ビジョン

## 2.1 三段階の事業進化

**第1形態（Y1-2）: アプリ** — 一本の優れたゲーム。「面白い」の証明と親の信頼獲得
**第2形態（Y3-5）: プラットフォーム** — 学校導入・検定・ルート工房で「コンテンツを他者が作る場」へ。先生が地元ルートを作り、自治体が観光ルートを出資し、供給が自走する
**第3形態（Y5-10）: 文化** — 「日本の子どもは、わらしべワールドでお金を覚えた」が普通の言葉になる状態。絵本・アニメ・カードゲーム・検定が相互送客する経済教育IPユニバース。海外はTrade Up Worldとして各国の地理・昔話で展開（エンジン共通・物語ローカル）

## 2.2 北極星の言い換え（10年版）
「週あたり交換成立数」の先にある究極KPIは——**「わらしべワールドで育った子が、大人になって最初にする金融行動の質」**。測定不能だが、全意思決定の羅針盤としてここに明記する（迷ったら「その機能は、10年後のあの子の判断を良くするか？」で判定）。

## 2.3 事業エンドゲームの選択肢（Kaiの三つの資本で整理）

| 選択肢 | 内容 | 評価 |
|---|---|---|
| A. 永久保有 | 低固定費×サブスク×学校の**キャッシュフロー資産**として持ち続ける | ◎ 本命。本作自体がKaiの「のれん分け投資」の実物。正社員ゼロ原則と整合 |
| B. 教育大手への事業売却 | 学研・ベネッセ等がY3-5に関心を持つ蓋然性は高い（実績×必修化の波） | ○ オプションとして常に磨いておく（そのためにも文書化・データ・監査可能な指標＝本スイートが企業価値そのもの） |
| C. IPライセンス分離 | アプリは保有、絵本・アニメ・海外はライセンスアウト | ○ 第3形態の現実的な資本効率解 |

**方針: Aを本線に、BとCの選択肢価値を落とさない経営**（=売れる状態を保ちつつ売らない）。

# 3. プリモーテム — 「1年後、失敗していた。なぜか？」

失敗したくない、に対する正面からの回答。起こりうる死因を先に全部殺す。

| # | 死因 | 兆候（早期警報） | 先回り策（対応文書） |
|---|---|---|---|
| 1 | 最初の15分が面白くなかった | テストで自発再プレイ<40% | Phase 0ゲートで進行停止・改修3回ルール・撤退基準（v2.0 §11-12）。**面白くないまま先に進むことを制度的に不可能にしてある** |
| 2 | 面白いのに発見されなかった | 指名検索が伸びない・DLの9割が知人 | 09の三山カレンダー＋チャレンジUGC＋フィーチャー申請＋受賞戦略。認知を開発と同格のバックログにする |
| 3 | 無料で満足され課金されない | D7は高いのに転換<2% | 転換の主役を「制限解除」でなく「親への価値（手帳・一言カード・自由研究）」に置く（08 C群）。親が払うのは子の制限解除ではなく**自分への効用** |
| 4 | 2週間で飽きられた | D7→D30の崖 | デイリー行商＋きせつの市＋クリア後アーク（本書§1）。「得る→与える」の動機転換が長期の器 |
| 5 | コンテンツ供給が追いつかない | ルート消化速度>制作速度 | 量産CI＋ルートエディタ＋UGC工房（第2形態）。供給を自分の労働から切り離す設計を最初から |
| 6 | 親の信頼を一度失う事故 | 不適切表現・データ問題の指摘 | 通信機能ゼロ・個人情報ゼロ・全確率開示の構造的安全（v2.0 §6）＋既知バグの誠実開示（06 §6） |
| 7 | 大手の模倣に速度負け | 類似アプリの発表 | 先行の学校実績・図鑑資産・信頼ブランドは後追い不能。むしろ市場が温まると解釈し、比較でなく独自機能（恩送り・手帳）で語る |
| 8 | 開発者のモチベーション枯渇 | 週次リズムの崩れ | 週90分運用（05 §8）・チケット駆動で「今日は何をすべきか」を考えない構造・Phase単位の完結感。**やる気を意志でなく仕組みで守る** |
| 9 | 教育効果を疑われ学校で止まる | 「エビデンスは？」で商談停止 | Phase 2で簡易プレポスト調査を実施し数値化（v2.0 §11-6）。監修者の顔と検定の級構造が信頼を補強 |
| 10 | 商形態とコンテンツ形式のミスマッチ（使い切り内容の月額売り）※外部レビューKS-1 | 月次解約>10%・初月解約集中 | 買い切りパック＋親価値サブスクの二層に改訂済み（v2.0 §7.1・決定27） |
| 11 | Phase 0ゲートの偽陽性通過 ※KS-2 | 判定指標がすべて合格なのにPhase 1のD7が沈む | 完走率の参考値化・テレメトリ判定・盲検化・n≥30に改訂済み（06 §5・決定30-31） |
| 12 | 認知計画の実行者不在 ※KS-3 | 09タスクの遅延累積・Kaiの週90分超過 | マーケ委託の体制化（v2.0 §10.1・決定32）。委託不在期間はKPI凍結 |

# 4. 成功の定義（三段階・正直版）

| レベル | 状態 | それでも得られるもの |
|---|---|---|
| 最低成功 | Phase 0-1で「子どもが夢中で遊ぶ」を実証、事業化は見送り | 実証済みゲームデザイン・本文書スイート・AI開発の完成された方法論（他事業へ全転用可能） |
| 標準成功 | 有料1万人・学校60校・単年黒字（v2.0財務のY3線） | 低固定費の恒常キャッシュフロー資産＋教育業界での信用 |
| 大成功 | カテゴリの代名詞・第2形態（プラットフォーム）移行・海外始動 | 文化としてのIP。「日本の子どもの金融リテラシーを一段上げた」という実績 |

**失敗の定義はただ一つ: 面白くないものを、面白くないと知りながら、大きくしてしまうこと。** それだけはゲート制度が防ぐ。

# 5. 最後に — この事業の一貫性

藁は、価値を信じて動かした者の手の中でだけ屋敷になる。
本事業も同じ構造でできている: 小さく検証し（Phase 0）、相手の困りごとを読み（親・学校・子それぞれの欲望リーディング）、価値の階段を一段ずつ登る（4段ロケット）。
**このアプリの作り方そのものが、このアプリの教えそのものである。** その一貫性こそが、長く愛されるプロダクトの条件だと考える。
